Ao criar um serviço, este precisa ser provido para aplicação, existem algumas formas de fazer isso:
  • Sendo declarado no modulo, onde o serviço ficará disponível para todos os componentes deste listados neste modulo;
  • Via root, onde o serviço se auto provê, assim podendo ser utilizado em toda a solução;
  • Via meta-dado providers dentro do @component do TS, ficando disponível apenas para um componente especifico.

@Input: Utiliza para que o componente pai consiga passar informações para o componente filho.

@Output: Utiliza para que o componente filho consiga passar informações para o componente pai.
